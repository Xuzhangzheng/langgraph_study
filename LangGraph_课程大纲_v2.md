> **说明**：本文件的优点（版本基线、硬约束、DoD、里程碑、22 课分阶段思路）已**合并入** `LangGraph_课程大纲.md`。本 `v2` 保留作**历史对照**（课号与仓库 `01`–`06` 脚本不完全一致，请勿作为唯一进度依据）。

## 课程说明

本大纲用于系统学习 LangGraph，目标是从基础概念逐步过渡到可上线的工程实践。  
课程遵循以下硬约束：

- 固定依赖版本学习，不随意升级接口
- 每课必须有“可运行示例 + 故障用例 + 最小回归验证”
- 课程演进以“可复现、可调试、可扩展”为优先

---

## 版本与接口基线（必须严格保持）

- Python: `3.11.3`
- `langgraph==1.1.10`
- `langchain==1.2.15`
- `langchain-openai==1.2.1`
- `python-dotenv==1.2.2`
- `ipykernel==7.2.0`

> 原则：后续所有示例、工具封装、状态结构、课程接口以当前版本能力为准，不引入破坏性接口变化。

## 分阶段课程路线（22 课）

### 阶段 A：图编排基础（第1-5课）

第1课：`Hello LangGraph` 最小可运行图  
内容：`StateGraph` 基本结构、`START/END`、节点函数输入输出、`invoke` 状态流转。  
产出：`01_hello_langgraph.py` 可运行最小图。

第2课：条件分支与路由（Branching / Routing）  
内容：`add_conditional_edges`、路由函数设计、状态驱动分支、默认分支策略。  
产出：`02_branching_graph.py`（含未知路由兜底分支）。

第3课：循环与重试（Loop / Retry）  
内容：循环回边、完成条件、`max_iterations` 安全退出、防止无限循环。  
产出：`03_loop_graph.py`（含失败重试与终止保护）。

第4课：Mini-Agent 工作流（分支 + 循环 + 自检）  
内容：任务分类、候选答案生成、评估反馈、失败重试与收敛。  
产出：`04_mini_agent_graph.py`（可观察迭代收敛过程）。

第5课：状态模型与合并机制（Reducer）  
内容：并行/多节点写入同一 state 字段时的冲突；`Annotated` reducer 设计。  
产出：一个“冲突复现 -> reducer 修复”的最小示例。  
验收：能解释“为什么同一个字段必须声明合并策略”。

### 阶段 B：模型与工具能力（第6-10课）

第6课：工具调用基础（Tool Node）  
内容：函数封装为工具节点、工具参数与返回协议、工具异常处理。  
产出：`05_tool_call_graph.py`（至少 2 个工具 + 1 个失败路径）。

第7课：LLM 接入与提示词分层  
内容：`langchain-openai` 接入、系统提示词/任务提示词分层、参数管理。  
产出：可配置模型与温度参数的调用封装。

第8课：消息状态与上下文管理  
内容：消息列表状态、历史裁剪策略、单轮到多轮状态演进。  
产出：支持多轮对话的 graph 版本。

第9课：结构化输出与解析防护  
内容：结构化输出约束、解析失败兜底、格式重试策略。  
产出：固定 schema 的输出节点（含解析失败回退）。

第10课：多工具路由与工具选择策略  
内容：多工具并存路由、工具选择器、回退策略、无法调用时兜底答复。  
产出：统一入口、自动选择工具的工具链路由图。

### 阶段 C：编排进阶（第11-15课）

第11课：子图（Subgraph）与模块化编排  
内容：复杂流程拆子图、子图复用、主图-子图接口设计。  
产出：至少 1 个复用子图模块。

第12课：并行分支与聚合（Fan-out / Fan-in）  
内容：并行执行、结果聚合、字段冲突合并策略、复杂度权衡。  
产出：并行检索/并行计算后汇总的 graph。

第13课：执行模式与运行控制  
内容：`invoke / stream / ainvoke / batch` 使用场景，`RunnableConfig` 注入（如 `thread_id`）。  
产出：同一图支持同步、流式、异步三种执行方式。

第14课：人机协同（Human-in-the-loop）  
内容：关键节点审批、中断与继续、人工纠错回流。  
产出：人工审核后继续执行的流程样例。

第15课：持久化与记忆（Checkpoint / Memory）  
内容：状态持久化、会话恢复、短期/长期记忆边界、可重放执行。  
产出：带 checkpoint 的可恢复会话示例。

### 阶段 D：工程化与治理（第16-20课）

第16课：可观测性与调试  
内容：节点级日志规范、状态快照、执行轨迹分析、定位方法。  
产出：统一日志字段规范 + 至少 1 条故障排查演示。

第17课：错误处理与鲁棒性工程  
内容：异常捕获、重试退避、降级策略、幂等性、超时控制。  
产出：标准错误处理模板（节点内/图级）。

第18课：测试与评测体系  
内容：节点单测、端到端回归、评测集构建、版本兼容性检查（接口不变）。  
产出：最小自动化回归脚本 + 基线评测报告。

第19课：RAG + LangGraph 整合  
内容：检索节点、重排节点、生成节点串联、失败回退路径。  
产出：可解释的检索增强问答图。

第20课：生产化部署与版本治理  
内容：分环境配置、依赖锁定、接口契约、上线检查清单、监控与回滚。  
产出：可部署清单与版本治理规则。

### 阶段 E：综合实战（第21-22课）

第21课：课程项目（Capstone）  
内容：从 0 到 1 构建可演示 LangGraph 应用（含工具、记忆、评测、日志、容错）。  
产出：完整项目与演示脚本。

第22课：复盘与进阶路线  
内容：知识图谱回顾、项目审阅、后续方向（高并发、复杂编排、企业级治理）。  
产出：个人进阶路线图（30/60/90 天）。

## 每课统一验收标准（DoD）

每节课结束必须满足：

1. 有可运行主示例（Happy Path）
2. 有故障/异常示例（Failure Path）
3. 有最小回归验证（可手工或脚本）
4. 有本课“接口不变”清单（输入字段、输出字段、状态字段）

## 里程碑检查点

- 里程碑 A（第5课后）：掌握图基础 + reducer，能稳定做条件和循环编排
- 里程碑 B（第10课后）：掌握 LLM + 工具调用，能做多工具路由型 Agent
- 里程碑 C（第15课后）：掌握可恢复执行、HITL、执行模式切换
- 里程碑 D（第20课后）：具备上线前工程治理能力
- 里程碑 E（第22课后）：完成完整项目并具备独立扩展能力
